# PPT质量检查清单

> 使用本清单确保PPT符合"高端"标准
> 建议：每完成一份PPT，逐项打勾检查

---

## 使用说明

1. **检查时机**：PPT初稿完成后
2. **检查方式**：逐页检查，每项打勾
3. **不合格处理**：标记问题页，修复后重新检查
4. **通过标准**：10项全部合格才算通过

---

## 质量检查清单（10项核心标准）

### ✓ 1. 逻辑性检查

**检查点**：
- [ ] 页面标题之间有递进关系（不是平行罗列）
- [ ] 遵循"金字塔原理"（结论先行）
- [ ] 章节划分清晰，每个章节有明确主题
- [ ] 最后一页呼应开头

**不合格示例**：
- ❌ 标题："我们的产品" → "我们的团队" → "我们的客户"（平铺，无逻辑）

**合格示例**：
- ✅ 标题："核心成果概览" → "业绩增长150%" → "新产品成功上线" → "未来规划"（有递进）

---

### ✓ 2. 数据准确性检查

**检查点**：
- [ ] 所有数字已与源数据核对
- [ ] 图表数据与文字说明一致
- [ ] 百分比加总为100%（饼图等）
- [ ] 增长率计算正确

**如何检查**：
1. 列出所有关键数字
2. 对照原始数据表格逐一核对
3. 用计算器验证增长率

**常见错误**：
- 图表显示Q4是95，文字说Q4是90
- 同比增长率算成了环比

---

### ✓ 3. 视觉一致性检查

**检查点**：
- [ ] 全篇使用同一套配色方案
- [ ] 标题字体、字号统一
- [ ] 正文字体、字号统一
- [ ] 页眉/页脚（如有）格式统一
- [ ] 图标风格统一（都是线性图标 或 都是面性图标）

**快速检查法**：
- 缩小视图到30%，浏览全部页面，看是否有"跳戏"的页面

---

### ✓ 4. 层次感检查

**检查点**：
- [ ] 每页有明显的主次关系（标题>数据>说明）
- [ ] 核心信息用大字号、粗体、主色强调
- [ ] 次要信息用小字号、浅色弱化
- [ ] 视觉焦点清晰（3秒内能看到重点）

**测试方法**：
- 站在3米外看屏幕，能否立即看到核心信息？

**不合格示例**：
- 标题24pt，正文也是24pt → 无层次

**合格示例**：
- 标题48pt + 数据60pt + 说明24pt → 层次清晰

---

### ✓ 5. 留白充足性检查

**检查点**：
- [ ] 页面空白区域≥30%
- [ ] 文字不贴边（四周边距≥页宽的10%）
- [ ] 元素之间有呼吸感（间距≥字号）
- [ ] 无拥挤感

**测试方法**：
- 用Photoshop的魔棒工具点击背景，选中区域应≥30%

**不合格示例**：
- 页面塞满文字和图表，无空隙

**合格示例**：
- 核心内容居中，四周大量留白

---

### ✓ 6. 渐变质量检查

**检查点**：
- [ ] 渐变过渡自然，无色阶断层
- [ ] 渐变方向一致（全篇都是135度 或 都是90度）
- [ ] 渐变不超过3种颜色
- [ ] 渐变色与配色方案一致

**常见问题**：
- Powerpoint默认渐变有色阶，需手动调整为"平滑"
- 一页用蓝色渐变，另一页用橙色渐变 → 不一致

---

### ✓ 7. 对齐严格性检查

**检查点**：
- [ ] 所有文本框左对齐或居中（不混用）
- [ ] 图表、图片与文字对齐网格线
- [ ] 标题位置统一（都在顶部同一高度）
- [ ] 使用PPT的对齐工具（不目测）

**检查方法**：
- 打开PPT的"网格线"和"参考线"功能
- 所有元素应吸附在网格上

---

### ✓ 8. 字号合理性检查

**检查点**：
- [ ] 最小字号≥20pt（投影展示标准）
- [ ] 标题≥48pt
- [ ] 核心数据≥60pt
- [ ] 正文24-28pt

**不合格示例**：
- 图表注释用16pt字号 → 投影时看不清

**特殊情况**：
- 如果是打印版，可适当缩小10-20%

---

### ✓ 9. 动画克制性检查

**检查点**：
- [ ] 每页动画不超过2个
- [ ] 动画仅用于引导视线或强调重点
- [ ] 动画速度适中（0.5-1秒）
- [ ] 无旋转、弹跳等花哨动画
- [ ] 如为静态展示（如PDF），可无动画

**推荐动画**：
- 标题：淡入（Fade In）
- 数据：擦除（Wipe）或放大（Zoom）
- 图表：逐条出现（Appear）

**禁用动画**：
- 旋转、弹跳、百叶窗等

---

### ✓ 10. 5秒可读性检查

**检查点**：
- [ ] 每页在5秒内能抓住核心信息
- [ ] 文字不超过7行（每行不超过20字）
- [ ] 图表简洁，无冗余元素
- [ ] 一页只讲一个核心观点

**测试方法**：
- 让同事看5秒，然后移开，问他记住了什么
- 如果回答不出核心信息，说明不合格

**优化方法**：
- 减少文字量（保留关键词）
- 增大核心信息字号
- 用颜色和位置突出重点

---

## 进阶检查（非必须，但能提升品质）

### 11. 配色和谐性
- [ ] 冷暖色对比适中
- [ ] 无刺眼的颜色组合
- [ ] 文字与背景对比度≥4.5:1（WCAG标准）

### 12. 图表专业度
- [ ] 无3D效果（除非特别需要）
- [ ] 无阴影效果（或仅用极淡的阴影）
- [ ] 坐标轴简洁，无冗余网格线
- [ ] 数据标签位置合理，不遮挡数据

### 13. 语言专业度
- [ ] 无错别字
- [ ] 无语病
- [ ] 数字格式统一（如"1000万"或"10,000,000"）
- [ ] 专业术语准确

### 14. 细节打磨
- [ ] Logo位置统一（如都在右下角）
- [ ] 页码格式统一
- [ ] 图片清晰度高（≥300dpi）
- [ ] 无拉伸变形的图片

## 团队协作建议

### 角色分工
1. **制作人**：负责内容和初稿
2. **审核人**：使用本清单检查
3. **美化师**（可选）：专门负责视觉优化

### 审核流程
```
制作初稿 → 自查（制作人用清单） → 提交审核 → 审核人检查 → 反馈修改 → 最终确认
```

### 重要汇报加强审核
- 重要汇报（如年终汇报、领导审阅）建议**双人审核**
- 第一轮：同事审核（专业性）
- 第二轮：领导预审（方向性）

## 常见问题

### Q：10项都要达到才算合格吗？
**A**：是的。这是"高端PPT"的最低标准。如果时间紧迫，至少保证前5项合格。

### Q：如何快速提升质量？
**A**：优先解决：
1. 数据准确性（避免致命错误）
2. 逻辑性（确保说得通）
3. 视觉一致性（看起来专业）

### Q：可以找AI帮忙检查吗？
**A**：可以！截图给Claude或GPT-4，使用 `04_AI提示词库/内容策划提示词.txt` 中的审阅提示词。
